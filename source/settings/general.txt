================
General Settings
================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

The general pane has the following setting options:

Set ``readOnlyMode``
--------------------

Limit Compass strictly to read operations, with all write and delete 
capabilities removed.

Enable MongoDB Shell
--------------------

Allow Compass to interacting with MongoDB deployments via the embedded shell.

Protect Connection String Secrets
---------------------------------

Hide credentials in connection strings from users.

Show Kerberos Password Field
----------------------------

Show a password field for Kerberos authentication. Typically only useful
when attempting to authenticate as another user than the current system user.

Set ``maxTimeMS`` for Database Operations
-----------------------------------------

Use the ``maxTimeMS`` parameter on the :guilabel:`General` pane to 
specify an upper time limit in milliseconds on all database operations
that run in |compass-short|.

To enable ``maxTimeMS`` for database operations:

.. procedure::
   :style: normal

   .. step:: On the :guilabel:`General` pane, enter a value in the 
      :guilabel:`Upper Limit for maxTimeMS for Compass Database Operations`
      value box. For a ``5`` second limit, enter ``5000``.

      .. figure:: /images/settings/settings-general-set-maxtime-ms.png
         :alt: Set maxtime ms

   .. step:: Click :guilabel:`Save`.

      .. figure:: /images/settings/settings-general-click-save.png
         :alt: Click save

      If your database operation goes over the time limit, 
      |compass-short| raises the following error:

      .. figure:: /images/settings/settings-general-time-limit-error.png
         :alt: Operation timeout error
